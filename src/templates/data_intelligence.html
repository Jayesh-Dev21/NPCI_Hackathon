User-Centric Z-Score Normalization Novel Approach

Traditional insider threat models treat all users identically. Our system normalizes behavioral features per user using Z-scores, detecting anomalies relative to each individual's baseline activity.

Why Z-Scores Matter

User A (developer) normally accesses 500 files/day. User B (executive) accesses 5 files/day. Without normalization, User A always triggers alerts. With Z-scores, we detect when User A accesses 1,000 files (2σ anomaly) or User B accesses 20 files (3σ anomaly).

Implementation

For each feature f and user u:
z_score = (f - μ_u) / σ_u

This captures deviations from personal baselines, making the model robust to role-based activity differences.

Impact on Performance

Z-score normalization reduced false positives by 68% in our ablation study, while maintaining 86% recall. Critical for production deployment in enterprise environments.
